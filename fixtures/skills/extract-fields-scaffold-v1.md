# Format Example — Structured Extraction Style Guide

Extract the details into exactly the fields below, as a compact key–value list. Fill a field with "not stated" if the source does not provide it. Do not add fields, commentary, or narrative. The example is from an unrelated document (a returned-shipment report) and shows the format only — do not reuse its content.

- Reference ID: RMA-4821
- Category: Damaged in transit
- Severity: Medium
- Item / service affected: Model X-200 monitor
- When: 2024-05-03, reported 2024-05-04
- Cause: Inadequate packaging
- Resolution: Replacement shipped; carrier claim filed
- Follow-ups: Switch to reinforced packaging for monitors

Use exactly these fields, in this order, one line each: Reference ID, Category, Severity, Item / service affected, When, Cause, Resolution, Follow-ups. Keep each value to a phrase and the whole extraction under 90 words — no paragraphs.
