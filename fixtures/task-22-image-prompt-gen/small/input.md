# Feature Requirements — "Focus Hours" for Acme Mail

## Problem
Knowledge workers open their inbox to handle one thing and lose the next 40 minutes to a flood of low-priority mail. They want to stay reachable for genuine emergencies without being pulled out of deep work by every newsletter, CC, and "just checking in."

## Requirements
1. A user can schedule recurring "Focus Hours" blocks (e.g., 9–11am daily) during which non-urgent mail is held silently.
2. During a block, only messages matching user-defined "let through" rules — named senders, the word "urgent", or direct replies to threads the user started — surface a notification.
3. Held mail is delivered in a single digest when the block ends, never as a drip of interruptions.
4. The user always sees a small, calm indicator that Focus Hours is on and how much time remains; turning it off is one tap and is never buried in settings.
5. Nothing is ever deleted, auto-archived, or permanently hidden — held mail is only delayed.

## Spirit
The feature should feel like a friendly gatekeeper, not a wall. Protective, calm, and reversible: the user stays fully in control and never fears missing something that truly matters.
